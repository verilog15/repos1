Binary data big-endian

Ce format est une image binaire brute des données de la mémoire dans un fichier.
Il n'y a pas d'en-tête, pas de commentaire, pas de codage. Le fichier n'est probablement pas éditable dans un éditeur de texte commun.
Dans le mode big-endian la mémoire est considérée comme une suite d'octet indépendamment de la dimension de ses mots, avec de gauche à droite les bytes de poids fort puis les bytes de poids faible.
Vous pouvez en trouvez des illustrations dans v3.0 hex bytes addressed big-endian

Binary data little-endian

Ce format est identique au précédent avec les mêmes caractéristiques.
Dans le mode little-endian la mémoire est considérée comme une suite d'octet avec de droite à gauche les bits de poids faible puis les bits de poids fort. Vous pouvez en trouvez des illustrations dans v3.0 hex bytes addressed little-endian
