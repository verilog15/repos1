Le format utilisé pour ce fichier image est intentionnellement simple. Cela vous permet d'écrire un programme, tel qu'un assembleur, qui génère des images mémoire qui peuvent ensuite être chargées. Par exemple si nous avions une mémoire comme ci-dessous :

Et ensuite toutes les valeurs sont à zéro. L'image résultante dans le fichier sera la suivante.

12 22 45 67 12*80  23 a2 bb
ce 4f 6a cc 45 7*cc 12 24

La première ligne identifie le format de fichier utilisé . Les valeurs suivantes, en hexadécimal, répertorient les données à partir de l'adresse 0. Les fins de ligne ne sont pas interprétées. Les valeurs dans ce fichier sont toujours présentées en hexadécimal (sans indication 0x).

Le fichier image peut utiliser un codage "run-length". Quand une valeur est répétée de multiple fois. Par exemple: 80 douze fois de suite est représenté par 12*80 . Notez que le nombre de répétitions est écrit en base 10.

Vous pouvez placer des commentaires dans le fichier en utilisant le symbole '#'. Tous les caractères dans la ligne à partir du symbole '#' seront ignorés par Logisim.

Si longueur de l'ensemble des données du fichier est plus courte que l'espace mémoire les cellules restantes seront initialisées à 0 pour les ROM et pour les RAM selon les paramètres dans Les options de projet, à 0 ou aléatoirement.

Si la largeur du mot mémoire en bit est inférieure à la largeur des mots de données du fichier. Les bits de poids fort en trop seront ignorés.
