Kontextmenüs und Dateien

Kontextmenüs

Abgesehen von den üblichen Punkten enthält das Kontextmenü für Speicherbauelemente vier weitere Optionen:

Inhalt bearbeiten...:
Öffnet den Hex-Editor um den Speicherinhalt zu bearbeiten.
Inhalt löschen :
Setzt alle Werte im Speicher auf 0.
Speicherabbild laden...:
Setzt alle Werte des Speichers auf die Werte aus einer Datei. Das Dateiformat ist weiter unten beschrieben.
Speicherabbild speichern...:
Speichert alle Daten aus dem Speicher in dem unten beschriebenen Dateiformat.

Die Dateiformate

  • v2.0 raw Historische Version von Logisim erlaubt Lauflängenkodierung-Kodierung.
  • v3.0 hex words plain Nahe an der wortstrukturierten Version von v2.0.
  • v3.0 hex words addressed Wie bisher mit der Möglichkeit, für jede Zeile der Datei eine Startadresse zu definieren.
  • v3.0 hex bytes plain big-endian
    v3.0 hex bytes plain little-endian In Byteströmen organisiert.
  • v3.0 hex bytes addressed big-endian
    v3.0 hex bytes addressed little-endian Wie bisher im Byte-Fluss organisiert mit der Möglichkeit, eine Adresse zu definieren.
  • Binary data big-endian
    Binary data little-endian Unkodiertes Speicherabbild.
  • Ascii byte escape big-endian
    Ascii byte escape little-endian ASCII-kodierte Datei mit Escape-Code, wie in C für nicht druckbare Zeichen.
